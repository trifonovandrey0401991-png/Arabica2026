# AUDIT CHECKLIST — Arabica2026
# Чек-лист проверки каждого модуля

## Flutter (клиент)
1. **setState без mounted** — вызов setState() без проверки `if (mounted)`
2. **Утечки контроллеров** — TextEditingController/ScrollController без dispose()
3. **Захардкоженные цвета** — цвета вместо AppColors
4. **Захардкоженные URL** — URL вместо ApiConstants
5. **Обработка ошибок** — пустые catch, проглоченные ошибки, пустые экраны при ошибке
6. **Null safety** — потенциальные null-краши (! без проверки, as без is)
7. **Логика ролей** — правильная фильтрация по ролям, нет доступа к чужим данным
8. **Утечки памяти** — незакрытые стримы, таймеры без cancel
9. **Формат телефона** — несовпадение форматов при поиске/сравнении
10. **UX проблемы** — нет индикатора загрузки, нет сообщения об ошибке

## Сервер (API)
11. **Авторизация** — маршруты без requireAuth/requireAdmin
12. **req.body.phone vs req.user.phone** — чтение телефона из тела вместо сессии (подмена)
13. **Двойная запись** — JSON + DB, нет пропуска одной из записей
14. **Время** — getHours()/new Date() вместо getMoscowTime()
15. **SQL инъекции** — конкатенация строк в SQL вместо параметров $1, $2
16. **Unbounded queries** — запросы без LIMIT, возврат всех записей
17. **Race conditions** — параллельные записи без блокировки
18. **writeJsonFile** — использование fsp.writeFile вместо writeJsonFile (атомарная запись)
19. **Валидация входных данных** — отсутствие проверки обязательных полей
20. **Обработка ошибок** — try/catch с пустым catch, 500 без логирования

## Общие
21. **Дублирование кода** — одинаковая логика в нескольких местах
22. **Мёртвый код** — неиспользуемые функции, импорты, переменные
23. **Несоответствие Flutter ↔ Server** — разные имена полей, разный формат данных
