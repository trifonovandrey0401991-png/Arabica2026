# Исправление ошибки выполнения PowerShell скрипта

## Проблема

```
Невозможно загрузить файл ... так как выполнение сценариев отключено в этой системе.
```

## Решение (выберите один из вариантов)

### Вариант 1: Использовать Batch файл (САМЫЙ ПРОСТОЙ) ✅

1. Скопируйте файл `run-sha-script.bat` на ваш компьютер
2. Дважды кликните на `run-sha-script.bat`
3. Скрипт запустится автоматически

### Вариант 2: Запустить через PowerShell с обходом политики

Откройте PowerShell и выполните:

```powershell
cd C:\Users\Admin\arabica2026
powershell -ExecutionPolicy Bypass -File .\get-sha-certificates.ps1
```

### Вариант 3: Разрешить выполнение скриптов для текущего пользователя

Откройте PowerShell **от имени администратора** и выполните:

```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

Затем запустите скрипт обычным способом:

```powershell
cd C:\Users\Admin\arabica2026
.\get-sha-certificates.ps1
```

### Вариант 4: Разрешить выполнение только для текущей сессии

В PowerShell выполните:

```powershell
Set-ExecutionPolicy -ExecutionPolicy Bypass -Scope Process
cd C:\Users\Admin\arabica2026
.\get-sha-certificates.ps1
```

## Рекомендация

**Используйте Вариант 1 (Batch файл)** - это самый простой и безопасный способ.

## Что означают политики выполнения

- **Restricted** - выполнение скриптов запрещено (по умолчанию)
- **RemoteSigned** - локальные скрипты выполняются, удаленные требуют подписи
- **Bypass** - выполнение всех скриптов разрешено (только для текущей сессии)

## Безопасность

- **Вариант 1 и 2** - безопасны, политика не изменяется
- **Вариант 3** - безопасен, разрешает только локальные скрипты
- **Вариант 4** - безопасен, действует только для текущей сессии PowerShell
