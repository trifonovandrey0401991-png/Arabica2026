## ДАННЫЕ — ПРОБЛЕМЫ СИНХРОНИЗАЦИИ

---

### Д-1. Удалённые заказы «воскресают» при переключении на JSON

Если `USE_DB_ORDERS=true` и заказ удалён из базы — JSON-файл остаётся. При переключении на JSON заказ появляется снова.

---

### Д-2. Баланс кассы хранится только в JSON

Таблица `main_cash` существует в базе, но `updateMainCashBalance()` в `withdrawals_api.js` пишет только в JSON-файлы. Финансовые данные без резервной копии в базе.

---

### Д-3. Задачи-сироты в базе данных

При удалении задачи из JSON, запись в базе НЕ удаляется. Со временем в базе накапливаются «мёртвые» задачи.
